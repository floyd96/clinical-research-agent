# 🩺 Clinic Research Assistant

An AI research assistant that searches **ClinicalTrials.gov** and **PubMed** simultaneously, delivering **grounded, cited** answers about clinical trials and published biomedical literature. Built as a Life Sciences demo — enterprise-grade UX, source-grounded responses, no fabrication.

---

## Features

### Intelligence
- **Single tool-capable agent** — every message runs through a LangGraph ReAct agent with live MCP tool access. The model decides when to call tools and when to answer from context; there is no brittle pre-classification step.
- **Grounded by design** — trial/paper field values come only from tool output. Missing fields render `Not specified`; the agent never invents NCT IDs, enrollment counts, eligibility, or results, and never asserts a superlative ("highest enrollment") about a value it hasn't retrieved.
- **Retrieval-aware follow-ups** — "which of those has the highest enrollment?" triggers a real query (sorted by `EnrollmentCount`) rather than a guess; pure follow-ups ("summarise the first one") answer from context with no tool overhead.
- **Adaptive output** — structured trial/paper cards for research; conversational prose for follow-ups, comparisons, and definitions.

### Research
- **Dual-source search** — ClinicalTrials.gov and PubMed via MCP (Model Context Protocol), independently or in parallel.
- **Source grounding** — NCT IDs and PMIDs are scraped directly from raw tool output, not generated by the model.
- **Structured cards** — trials (NCT ID, status, phase, enrollment, condition, intervention, sponsor, location) and papers (PMID, authors, journal, study type, key finding).
- **Inline citations** — a `📎 Sources` footer with clickable ClinicalTrials.gov / PubMed links at the bottom of each response.
- **Live tool status** — each database call surfaces as a step badge (🏥 / 📚) as it happens.

### UI & UX
- **Live token streaming** — responses stream token-by-token.
- **Personalised greeting** — time-of-day greeting with the signed-in user's real name (from Google profile).
- **Sound effects** — send and done cues via the Web Audio API (zero dependencies).
- **Randomised starters** — the welcome screen rotates 4 questions from a curated pool each session.
- **Mayo-branded theme** — deep-blue header, Inter typography, custom CSS/JS.

## Architecture

```
User message
     │
     ▼
LangGraph ReAct agent  (gpt-4o-mini + MCP tools)
     │   model decides per the system prompt:
     ├── needs data?  → call ClinicalTrials.gov / PubMed tools (in parallel when both apply)
     └── answer in context?  → respond directly, no tool call
     │
     ▼
Stream tokens → scrape NCT/PMID sources → append inline citations → persist to Supabase
```

A single agent instance is created **once per server lifetime** (`_shared_agent` in `app.py`) — MCP tools are fetched once, not per session, eliminating per-reload latency. Conversation state is kept in `lc_messages` (last 6 messages) in the Chainlit user session. Synchronous Supabase writes run via `asyncio.to_thread` so they never block the event loop.

The system prompt (`prompts.py`) is assembled from independently editable sections: identity, scope, follow-up handling, tool selection, compliance, **grounding/anti-fabrication**, and output format.

## Tech Stack

| Layer | Technology |
|---|---|
| LLM | OpenAI **gpt-4o-mini** (`langchain-openai`) |
| Agent framework | LangGraph `create_react_agent` (ReAct loop with MCP tool binding) |

| Auth | Google OAuth (beta whitelist) |
| Persistence | Supabase (Postgres) |

## Local Setup

**Prerequisites:** Python 3.13, an OpenAI API key, a Supabase project, and Google OAuth credentials.

```bash
# 1. Clone
git clone https://github.com/metafloyd/clinical-research-agent.git
cd clinical-research-agent

# 2. Virtual environment
python -m venv .venv
.venv\Scripts\activate        # Windows
# source .venv/bin/activate   # macOS / Linux

# 3. Install dependencies
pip install -r requirements.txt

# 4. Configure .env (see below)

# 5. Run
.venv\Scripts\python.exe -m chainlit run app.py --port 8504
```

### Environment variables (`.env` locally / Render dashboard)

```
OPENAI_API_KEY
SUPABASE_URL
SUPABASE_SERVICE_ROLE_KEY
OAUTH_GOOGLE_CLIENT_ID
OAUTH_GOOGLE_CLIENT_SECRET
CHAINLIT_AUTH_SECRET
CHAINLIT_URL=https://clinical-research-agent.onrender.com
```

> Google OAuth redirect URIs must include both the local callback
> (`http://localhost:8504/auth/oauth/google/callback`) and the production one.

### CLI Runner

A standalone command-line interface tests the agent directly without the Chainlit UI:

```bash
.venv\Scripts\python.exe research_agent.py
```

## Deployment (Render)

1. Push to GitHub (`main` auto-deploys).
2. Build: `pip install -r requirements.txt` · Start: `bash start.sh` → `chainlit run app.py --port $PORT --host 0.0.0.0`.
3. Set the environment variables above in the Render dashboard.
