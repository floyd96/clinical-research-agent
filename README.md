# 🧬 Clinical Research Intelligence Agent

An AI-powered research assistant that searches **ClinicalTrials.gov** and **PubMed** simultaneously, delivering grounded, cited answers about clinical trials and published biomedical literature.

---

## Features

### Intelligence
- **LLM intent classifier** — every message is classified as a new research query or a follow-up before any tools are called, using a dedicated Qwen 72B call capped at 5 tokens
- **Two-path routing** — new queries go through the full ReAct agent with live tool calls; follow-ups go directly to the model with clean conversation history, no tool overhead
- **Adaptive response format** — structured trial/paper cards for new research; conversational prose for follow-ups, comparisons, and summaries
- **Session-wide context memory** — the classifier indexes all retrieved NCT IDs, PMIDs, and topic headings across the full session, so follow-up detection stays accurate regardless of response length

### Research
- **Dual-source search** — queries ClinicalTrials.gov and PubMed via MCP (Model Context Protocol), independently or together
- **Source grounding** — NCT IDs and PMIDs are extracted directly from raw tool outputs, not generated by the model
- **Structured output** — trial cards (NCT ID, phase, status, eligibility highlights) and paper cards (PMID, authors, journal, key finding)
- **Live tool status** — real-time display of which database is being queried as it happens
- **Response limits** — maximum 3 trials and 3 papers per response for speed and focus

### UI & UX
- **Token streaming** — responses appear in real time with a live cursor
- **Session persistence** — conversation survives page refreshes via localStorage (per browser)
- **Session export** — download the full session as a styled HTML report, printable to PDF via browser
- **Copy response** — strips markdown and copies plain text to clipboard
- **Sound effects** — send whoosh and done chime via Web Audio API (zero dependencies)
- **Randomised example questions** — welcome screen rotates 4 questions from a pool of 16 per session
- **Mobile responsive** — breakpoints at 480 / 580 / 640 / 768px, light theme forced on mobile

## Architecture

```
User message
     │
     ▼
Intent Classifier (Qwen 72B, max 5 tokens)
     │
     ├── "followup"  → run_direct()    ← model.astream(), clean chat history, no tools
     │
     └── "research"  → run_agent()     ← ReAct agent, MCP tools, live status bar
```

**Two message histories are maintained in parallel:**

| History | Contains | Used by |
|---|---|---|
| `lc_messages` | Full agent history including ToolMessages and tool call artifacts | `run_agent()` — agent needs prior tool outputs to avoid re-fetching |
| `chat_display` | Clean human/assistant text pairs as displayed to the user | `run_direct()`, localStorage persistence, HTML export, UI rendering |

## Tech Stack

| Layer | Technology |
|---|---|
| LLM | Qwen/Qwen2.5-72B-Instruct via HuggingFace Serverless Inference API |
| Agent framework | `langchain.agents.create_agent` (ReAct loop with MCP tool binding) |
| Intent classification | Same Qwen 72B model, separate endpoint capped at 5 output tokens |
| Data sources | ClinicalTrials.gov MCP · PubMed MCP (third-party hosted) |
| UI | Streamlit |
| Session persistence | `localStorage` via `streamlit-javascript` |
| Hosting | Streamlit Community Cloud (auto-deploys on `git push`) |

## Local Setup

**Prerequisites:** Python 3.10+, a [HuggingFace account](https://huggingface.co) with a token that has access to Qwen 2.5-72B-Instruct serverless inference.

```bash
# 1. Clone the repo
git clone https://github.com/metafloyd/clinical-research-agent.git
cd clinical-research-agent

# 2. Create and activate a virtual environment
python -m venv .venv
.venv\Scripts\activate        # Windows
# source .venv/bin/activate   # macOS / Linux

# 3. Install dependencies
pip install -r requirements.txt

# 4. Add your HuggingFace token
cp .env.example .env
# Edit .env → set HF_TOKEN=hf_your_token_here

# 5. Run the Streamlit app
streamlit run research_app.py --server.port 8503
```

### CLI Runner

A standalone command-line interface is also available for testing the agent directly without the Streamlit UI:

```bash
python research_agent.py
```

## Deployment (Streamlit Community Cloud)

1. Push to GitHub
2. Go to [share.streamlit.io](https://share.streamlit.io) → New app
3. Select the repo, set main file to `research_app.py`
4. In **Advanced settings → Secrets**, add:
   ```toml
   HF_TOKEN = "hf_your_token_here"
   ```
5. Deploy — auto-redeploys on every `git push` to main
